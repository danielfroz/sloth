# Sloth 0.3.0

_Unreleased_

Sloth 0.3.0 **slims the DI container**, makes **`inject` lazy by default** (so
circular dependencies and init ordering stop being a thing), turns **`warmup()`**
on by default for fail-fast startup, and adds a **cohesive bootstrap** —
`@Repository`/`@Service` discovery plus an ordered `Initializer` pipeline. For most
services the upgrade is a version bump plus, optionally, adopting the new bootstrap.

## ✨ Highlights

### `inject` is lazy by default — cycles & ordering just disappear

`DI.inject(token)` now branches on how the token is registered:

- **class** dependency → a transparent proxy constructed on **first use**, so two
  services can depend on each other with **no annotation**;
- **value / factory** dependency → resolved **eagerly** (so primitives — secret
  strings, URIs — work; a proxy can't wrap a primitive);
- **unregistered** token → throws at inject time (i.e. at boot under `warmup`).

```ts
class A {
  constructor(private readonly b = DI.inject(Types.B)) {} // plain inject
  run() { return this.b.help() }                          // B built here, on first use
}
class B {
  constructor(private readonly a = DI.inject(Types.A)) {} // mutual — nothing special
}
```

There is **no separate `DI.lazy`** anymore — `inject` is the lazy one. Methods are
bound to the real instance (private `#fields` and fluent `return this` work; bound
methods are cached). Cost: object deps are proxies → a tiny per-access hop and
`dep instanceof Class` is `false`; values/primitives are unaffected.

### `warmup()` — on by default

`start()` now runs `warmup()` automatically (opt out with `{ warmup: false }`),
eagerly constructing every registered class/factory token so a missing or
misconfigured dependency fails at **boot**, not on a request. Because `inject`'s
*lookup* is eager, lazy deps don't hide wiring mistakes.

```ts
await app.start({ port: 3000 })                // warmup runs automatically
await app.start({ port: 3000, warmup: false }) // opt out
```

### Cohesive bootstrap — `@Repository`/`@Service` + `Initializer`

Stop hand-wiring `inits/*`. Discover order-free class bindings; run the
order-sensitive I/O bootstrap as an explicit, typed pipeline.

```ts
@Repository(Types.Repos.Order)
export class OrderMongo implements OrderRepository { /* ... */ }

export class SecretInit implements Initializer {
  async init() { container.register(Types.SecretClient, { useValue: new SecretClient(api) }) }
}

// main.ts
import '@/repositories/mongo/index.ts'   // run @Repository decorators
import '@/handlers/cqrs/index.ts'         // run @Route decorators

const app = new Application({ framework: new OakFramework() })
app.Providers.discover()                                  // register @Repository/@Service
await app.Inits.run(LogInit, SecretInit, MongoInit, ApiInit, EventsInit) // ordered I/O
app.Handlers.pipeline({
  before: [HealthMiddleware],
  after: [NotFoundMiddleware]
})
await app.start({ port: 3000 })                           // warmup → listen
```

For decorator-free/one-off bindings use `container.register(...)` directly. The
old dormant `ServiceBuilder` is replaced by `app.Providers`.

### A much smaller container

The DI module (originally a di-wise fork) dropped from **18 files / ~1,130 lines**
to **7 files / ~420**, keeping only what Sloth and its services actually use:
`Type` tokens, `Scope`, `register`/`resolve`/`inject`/`has`/`warmup`. Resolution is
a synchronous core (ambient current-container + a `Set` for circular detection).

## ⚠️ Breaking changes

### Default scope is now `Singleton`

`container.register(token, { useClass })` without an explicit `{ scope }` defaults
to **`Singleton`** (was `Transient`), aligning it with `Controller.add` / `@Route`
/ `app.Providers`. Safe for the standard pattern (anything injected into a Singleton
handler was already shared across concurrent requests). Opt back with
`{ scope: DI.Scope.Transient }`.

### `inject` returns proxies for class deps

A class dependency is now a lazy proxy, so `injectedDep instanceof SomeClass` is
`false` and object identity differs. Method calls and field access are transparent.
Value/factory deps are unchanged (eager, real instance).

### Removed (unused) DI surface

None of these were used by any service or the framework:

| Removed | Replacement |
|---|---|
| standalone `DI.lazy` | `inject` is lazy by default |
| `@Injectable` / `@Scoped` / `@AutoRegister` / `@Inject` / `@InjectAll` | `@Repository`/`@Service` + `app.Providers.discover()`, or `container.register(...)` |
| `injectAll` / `InjectAll` / `container.resolveAll` | resolve a single token |
| `injectBy` / `inject.by` / `Injector` token | lazy `inject` |
| multi-token `inject(a, b, …)` | inject one concrete token |
| `container.createChild()` | `createContainer()` |
| container middleware (`applyMiddleware`, `resolveAllSafe`) | — |
| `Build(...)` / `Value(...)` builder tokens | `{ useFactory }` / `{ useValue }` |
| `Type.inter` / `Type.union` | separate tokens |

## 🆕 New API

`@Provide` / `@Repository` / `@Service`, `registerProviders` / `clearProviders`,
`Initializer`, `app.Providers` (`discover`), `app.Inits`
(`run`). An unregistered **class** passed to `container.resolve` constructs as
Transient (di-wise behavior — services resolve event-handler classes this way).

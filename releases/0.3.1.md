# Sloth 0.3.1

_Released 2026-06-21_

A docs/maintenance patch — **no API or behavior changes** (the published package is
identical to 0.3.0).

## Notes

- Documents the **JSR explicit-return-type requirement** in the project guide:
  every exported (public-API) function must have an explicit return type, or JSR's
  `publish` slow-types check fails. (This is what blocked the 0.3.0 publish, via the
  `Provide()` decorator factory — now fixed and shipped in 0.3.0.)
- Adds `deno publish --dry-run --allow-dirty` as a **required pre-push validation**
  step (it runs the same slow-types/publish checks as the workflow), so the issue is
  caught locally before a push to `main`.
- README: the bundled-example references now link to
  [examples/oak](https://github.com/danielfroz/sloth/tree/main/examples/oak) and
  [examples/express](https://github.com/danielfroz/sloth/tree/main/examples/express).

## Install

```shell
deno add jsr:@danielfroz/sloth@0.3.1
```

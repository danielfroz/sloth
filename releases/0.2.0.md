# Sloth 0.2.0

_Released 2026-06-21_

Sloth 0.2.0 makes endpoints and middleware **declarative and low-boilerplate**,
adds a structured request pipeline, and upgrades the Express adapter to v5.
Almost everything is **opt-in and backward compatible** — `Controller.add()` and
`app.Handlers.push()`/`routes()` still work — so you can adopt the new patterns
gradually.

## ✨ Highlights

### `@Route` discovery — declare endpoints on the handler, no `controllers/*.ts` to maintain

```ts
@Route('/echo/save')
export class EchoSaveHandler implements CommandHandler<EchoSaveCommand, EchoSaveCommandResult> { ... }
```

At startup the controllers are assembled automatically from every
`@Route`-decorated handler.

### Structured global pipeline — declare the whole request order in one call

```ts
app.Handlers.pipeline({
  before: [ LogMiddleware ],      // run before every controller
  after:  [ NotFoundMiddleware ], // catch-all, runs last
})
```

`before → controllers → after`. `pipeline()` is sugar over `push()`/`routes()`.

### Per-route (scoped) middleware — auth on writes, public reads

```ts
@Route('/echo/save', { use: [AuthMiddleware] })   // scoped to this endpoint only
```

Also works in the manual style:

```ts
Controller.add({ ..., middlewares: [AuthMiddleware] })
```

### Express 5

The Express adapter now targets `express@5` (and `multer@2`).

## ⚠️ Breaking changes

- `Errors.AuthError(code, message)` is now 2-arg (the 3-arg `description` form was removed).
- `Errors.CodeDescriptionError` → use `Errors.CodeError(code, message)`.

Both are mechanical find-and-replace. Everything else is additive.

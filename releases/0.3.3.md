# Sloth 0.3.3

_Released 2026-07-15_

A bug-fix patch for the **Oak** framework's multipart parser — **no API changes**.
Large file uploads (`multipart/form-data`) crashed with
`RangeError: Maximum call stack size exceeded`; they now upload at any size.

## Fix

- **`OakFramework` multipart base64 no longer overflows the stack.** The parser
  encoded each uploaded file with `btoa(String.fromCharCode(...new Uint8Array(buffer)))`.
  The spread passes every byte as a separate function argument, so once a file
  exceeds the JS engine's argument-count limit (~64 KB in V8/Deno) the call stack
  overflows and the request dies with a 500 `service.error` /
  `Maximum call stack size exceeded`. Encoding now runs in fixed 32 KB chunks
  (`toBase64`), so no `String.fromCharCode` call ever gets an unsafe argument count.

  Symptom before: small images uploaded fine, larger ones (logos/covers) 500'd —
  the failure scaled with file size.

- **Express framework unaffected.** Its parser already uses `Buffer.toString('base64')`
  (native, no spread) — no change needed.

## Compatibility

Backward compatible. Same base64 output, only the encoding path changed. Any
`multipart/form-data` route (photo/document uploads) that failed on large files now
succeeds; small-file behavior is unchanged.

## Install

```shell
deno add jsr:@danielfroz/sloth@0.3.3
```

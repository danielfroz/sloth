# Sloth 0.3.2

_Released 2026-07-10_

An `ApiFetch` patch that makes outbound API failures **diagnosable** — no breaking
changes. Previously every `get`/`post` failure (a dead upstream, a timeout, a
connection reset, a DNS error, **and** a real upstream 500) collapsed into a single
opaque `ApiError(status: 500, code: 'service')`, so operators could not tell "the
service is down" from "the service returned an error". Now the thrown `ApiError`
names the actual failure.

## Changes

- **Granular network-error classification.** A `fetch` rejection is inspected
  (error name, `error.cause`, and message — across Deno and Node/undici) and mapped
  to a gateway-appropriate status + dotted code:

  | Failure | `code` | `status` |
  |---|---|---|
  | connection refused | `connection.refused` | 503 |
  | host/network unreachable | `network.unreachable` | 503 |
  | timeout (explicit or OS-level) | `connection.timeout` | 504 |
  | connection reset | `connection.reset` | 502 |
  | connection aborted | `connection.aborted` | 502 |
  | DNS lookup failure | `dns.notfound` | 502 |
  | other network failure | `connection` | 502 |

  The `ApiError.message` preserves the original driver text (`reason: <raw>`) for logs.
  An upstream that **did** respond keeps its **real** `res.status` and coded error
  body (`json.error.code`) — it is never reclassified.

- **Response-body parse errors are distinguished.** A non-JSON body no longer hides
  behind the generic 500: a non-2xx non-JSON body → `code 'service'` with the raw
  body text at its real status; a 2xx non-JSON body →
  `ApiError(status: 502, code: 'response.invalid')`.

- **Opt-in request timeout.** New `timeout?: number` (ms) on both
  `ApiInitOptions` (client-wide) and `ApiGetOptions`/`ApiPostOptions` (per-request
  override), enforced via `AbortSignal.timeout`. Unset ⇒ **no** timeout (unchanged
  behavior); a fired timeout throws `504 / connection.timeout`.

- Internal: `get`/`post` now share a private `_request` helper (removed the
  duplicated `new Promise(async …)` bodies).

## Compatibility

Backward compatible. Network failures that used to surface as `status 500` now
surface as `502/503/504` (verbatim to the HTTP client via the Oak/Express adapters);
no existing code branches on the old `500`/`'service'` pair. `throwOnError` remains
unused, as before.

## Install

```shell
deno add jsr:@danielfroz/sloth@0.3.2
```
